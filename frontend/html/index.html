// 01
Reproducibility
Identical inputs always produce identical outputs and hashes — across runs, machines, and time.
// 02
Tamper detection
Any change to inputs triggers a visible hash divergence with an exact diff pointing to the root cause.
// 03
Full observability
Every step is traced with duration, type, and output. No black-box AI decisions, ever.
02 architecture — step types _
// step.type = "local"
tool() — local execution
Pure Python callables with no side effects. Seeding, card draws, data transforms — always deterministic, zero latency, zero cost.
// step.type = "llm"
llm() — cached inference
LLM steps are keyed by prompt hash + model name. On repeat runs the response is served from cache — identical output, near-zero cost, <10ms.
// step.type = "condition" | "parallel"
condition() + parallel()
Branch on step outputs, run independent steps concurrently. Complex workflows without sacrificing traceability or hash integrity.

04 use cases — regulated industries _
// finance
Loan scoring
Regulators can replay any credit decision with the original inputs and verify the output hash — proving no model drift or data tampering between runs.
// insurance
Claims assessment
Every claim evaluation is a frozen, replayable trace. Disputes resolved by running the exact program — not by trusting logs that could have been altered.
// healthcare
Diagnostic support
Clinical AI recommendations are reproducible artifacts. Any change in patient data is flagged immediately, protecting against silent upstream updates.
